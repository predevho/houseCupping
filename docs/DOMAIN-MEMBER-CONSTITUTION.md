# DOMAIN-MEMBER-CONSTITUTION

## 회원 도메인 핵심 원칙

### 1. 인증 위임
- 인증(Authentication)은 Supabase Auth에 완전히 위임한다.
- 비밀번호 해싱, 세션 관리, 토큰 갱신을 직접 구현하지 않는다.

### 2. 프로필 자동 생성
- 사용자가 회원가입하면 `profiles` 레코드가 자동으로 생성된다 (DB 트리거).
- 프론트엔드에서 프로필 생성 로직을 별도로 구현하지 않는다.

### 3. 프로필 = 공개 정체성
- `profiles` 테이블은 앱 내 사용자의 공개 정체성을 나타낸다.
- 민감 정보(이메일, 전화번호 등)는 `auth.users`에만 저장하고 `profiles`에 복사하지 않는다.

### 4. 세션 관리
- 세션은 서버사이드 쿠키 기반으로 관리한다 (`@supabase/ssr`).
- 미들웨어에서 모든 요청에 대해 세션을 갱신한다.

### 5. 회원 탈퇴
- 탈퇴 시 `auth.users` 삭제 → `profiles` CASCADE 삭제 → 커핑 노트·평점·좋아요·댓글 CASCADE 삭제된다.
- 단, 원두(`beans`)는 공유 자산이므로 탈퇴 후에도 보존되고 `user_id`만 NULL로 전환된다.
