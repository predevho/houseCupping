# DOMAIN-BEAN-CONSTITUTION

## 원두(Bean) 도메인 핵심 원칙

### 1. 원두 레코드는 사용자가 등록한 것
- 원두 레코드는 사용자가 직접 경험한 원두를 등록한 것이다.
- 등록과 수정은 등록자에게 허용하되, 삭제는 admin만 가능하다.
- 사용자는 admin에게 삭제를 요청하는 방식으로 처리한다.

### 2. 카페 단위 등록
- 원두는 특정 카페(`cafe_name`)에서 구매/제공받은 것으로 등록된다.
- 동일한 원두라도 카페가 다르면 별개의 레코드로 관리한다.

### 3. 정보의 선택성
- 원두의 필수 정보는 `cafe_name`, `bean_name` 뿐이며, 나머지는 모두 선택 사항이다.
- 스페셜티 업체마다 원산지, 품종 등 정보 제공 수준이 다를 수 있으므로 정보가 없어도 등록을 막지 않는다.

### 4. 원두와 커핑 노트의 관계
- 하나의 원두에 여러 커핑 노트를 작성할 수 있다 (1:N).
- 사용자가 원두를 삭제할 수 없으므로, 다른 사용자의 커핑 노트가 의도치 않게 사라지지 않는다.
- admin이 원두를 삭제할 경우 연결된 커핑 노트도 함께 삭제된다 (CASCADE). 이는 admin의 의식적인 판단 하에 이루어진다.
