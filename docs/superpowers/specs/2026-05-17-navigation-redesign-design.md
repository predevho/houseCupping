# 네비게이션 UI 리디자인 Design Spec

**Date:** 2026-05-17
**Status:** Approved

---

## Goal

현재 단순한 텍스트 헤더를 `모던 제품 느낌`의 내비게이션 구조로 리디자인한다.
현재 웹앱은 데스크톱 중심 정보구조를 우선 정리하고, 모바일은 별도 앱 프로젝트로 확장하기 전까지 `깨지지 않는 최소 반응형`만 제공한다.
따라서 이번 범위에서는 `상단 nav + 좌측 카테고리 nav + 메인 콘텐츠` 3단 구조를 데스크톱 중심으로 구현하고, 모바일 전용 nav 컴포넌트는 만들지 않는다.

## Design Direction

### 비주얼 톤

- 전체 인상: 밝고 정돈된 SaaS/Product UI
- 배경: 화이트와 매우 옅은 쿨그레이 계열
- 포인트 컬러: 기존 버건디 계열을 유지하되, `강조 버튼/현재 위치`에만 선별적으로 사용
- 경계 표현: 진한 그림자보다 얇은 border와 약한 blur/배경 대비 사용
- 텍스트 위계: 진한 네이비/차콜 텍스트 + 보조 설명은 중간 회색

### 구조 원칙

- 상단 nav는 “전역 액션”
- 좌측 nav는 “서비스 카테고리”
- 메인 콘텐츠는 피드/리스트 중심
- 시각적 강조는 배경색보다 spacing과 active state로 만든다

## Architecture

### 데스크톱 구조

```text
┌──────────────────────────────────────────────────────────────────────┐
│ House Cupping            Search / CTA 영역            로그인 / 프로필 │
├────────────────┬─────────────────────────────────────────────────────┤
│ 홈             │                                                     │
│ 원두           │               메인 콘텐츠 영역                      │
│ 커핑 노트      │                                                     │
│ 프로필         │               홈 피드 / 목록 / 상세                 │
│                │                                                     │
└────────────────┴─────────────────────────────────────────────────────┘
```

- 상단 nav: 전체 폭
- 좌측 nav: 고정 폭 패널
- 메인 영역: 현재 페이지 콘텐츠 렌더

### 모바일 구조

```text
┌─────────────────────────────────────────────┐
│ House Cupping                  로그인/프로필 │
├─────────────────────────────────────────────┤
│                                             │
│             메인 콘텐츠 영역                 │
│                                             │
└─────────────────────────────────────────────┘
```

- 좌측 nav 숨김
- 상단 nav만 유지
- 메인 콘텐츠가 좁은 화면에서도 깨지지 않도록 spacing과 width만 최소 조정

## Navigation Model

### 상단 nav

상단 nav는 아래 항목만 담당한다.

- 좌측: 브랜드 로고/서비스명
- 우측:
  - 비회원: `로그인`, `회원가입`
  - 로그인 사용자: `프로필`, `로그아웃`

이번 범위에서는 검색창이나 알림 UI는 넣지 않는다.

### 좌측 카테고리 nav

첫 버전 카테고리는 아래 4개만 사용한다.

- `홈` → `/`
- `원두` → `/beans`
- `커핑 노트` → `/cupping`
- `프로필` → `/profile`

### 활성 상태

- 현재 경로와 일치하는 항목은 강조
- 강조 방식:
  - 데스크톱: 옅은 배경 + 좌측 포인트 bar + 진한 텍스트
  - 모바일: 이번 범위에서는 좌측 nav를 숨기므로 상단 nav의 계정 액션만 유지

## Component Structure

```text
AppLayout
  ├── TopNav (Header.tsx 확장 또는 분리)
  ├── SideNav (신규)
  └── Main content
```

### 권장 분리

- `Header.tsx`: 상단 브랜드/계정 액션만 담당
- `SideNav.tsx`: 데스크톱 카테고리 nav
- `(app)/layout.tsx`: 데스크톱 우선 레이아웃 배치 담당

이렇게 나누면 계정 상태 분기와 카테고리 분기를 섞지 않아도 되고, 나중에 모바일 앱 프로젝트에서 별도 네비게이션 구조를 설계하기도 쉽다.

## Layout Specification

### 데스크톱

- 상단 nav 높이: `56px ~ 64px`
- 전체 래퍼: `max-w-7xl` 또는 이에 준하는 넓은 컨테이너
- 좌측 nav 폭: `200px ~ 220px`
- 메인 콘텐츠: `min-w-0 flex-1`
- 홈 피드/목록은 중앙에 좁게 두지 말고, 충분히 넓게 읽히도록 배치

### 모바일

- 상단 nav 높이: `56px`
- 좌측 nav: 숨김
- 메인 콘텐츠: 가로 스크롤이 생기지 않도록 `px-4` 수준의 최소 패딩 유지
- 상단 nav만으로 기본 이동 진입을 제공

## Behavior

### 비회원

- 상단 우측에 `로그인`, `회원가입`
- `/profile` 링크는 좌측 nav에 보이지만, 클릭 시 기존 보호 라우트 흐름대로 `/auth?next=/profile`
- 홈/원두/커핑 노트는 자유롭게 열람 가능

### 로그인 사용자

- 상단 우측에 프로필 링크와 로그아웃 버튼
- 데스크톱에서는 좌측 카테고리 nav 사용
- 모바일에서는 상단 nav만 유지

## Testing Scope

- `Header`가 비회원/로그인 상태에 따라 올바른 상단 액션을 보여주는지
- `SideNav`가 현재 경로 기준 active state를 렌더링하는지
- `(app)/layout.tsx`가 데스크톱에서는 Header + SideNav 구조를 포함하고, 작은 화면에서는 SideNav를 숨기는지

| `FE/src/components/layout/__tests__/SideNav.test.tsx` | 신규 | active state 테스트 |
| `FE/src/app/globals.css` | 수정 | 필요한 전역 토큰/보조 색상 정리 |

## Scope Exclusions

- 검색창
- 알림/북마크/설정 메뉴
- 모바일 전용 웹 네비게이션 컴포넌트
- 접히는 데스크톱 사이드바
- 다크 모드
- 관리자 전용 nav 분기
